Streams

Streaming data means a constant flow of events, each containing information about a change of some kind. Streaming data reflects how modern businesses actually work—thousands of small changes happening all day, every day. Spring’s ability to handle events and easily build applications around them lets your apps stay in sync with your business.

What is Streaming Data?

Streaming data means a constant flow of data about an event (or “change of state”). One example might be a stock ticker. Every time a stock price changes, a new event is created. It’s called “streaming data” because there are thousands of these events resulting in a constant stream of data.

Streaming Data and Microservices

When combined with microservices, event streaming can open up exciting opportunities—event-driven architecture being one common example. Spring can simplify the production, processing, and consumption of events, providing several useful abstractions.

Spring Cloud Stream

Spring Cloud Stream improves your productivity when working with Apache Kafka, RabbitMQ, Azure Event Hub, and more. It features three simple abstractions that simplify your code. “Binders” provide integration with the external messaging systems. “Bindings” bridge the gap between the messaging systems and your code. “Messages” provide a simple structure that your code can use to communicate.

Spring Cloud Stream also provides support for provisioning, content conversion, error handling, configuration management, consumer groups, partitioning, monitoring, and health checks.

Spring Cloud Kafka Streams

Spring Cloud Stream provides a second, more specific binder solely for working with Kafka Streams. This special binder still focuses on developer productivity but adds support for Kafka-specific features like KStream, KTable, and GlobalKTable. As with regular Spring Cloud Stream, the binder also takes care of connecting to Kafka, as well as creating, configuring, and maintaining the streams and topics. Learn more

Spring Cloud Function

With Spring Cloud Function, you can write functions once and run them anywhere (AWS, Azure, etc.) while continuing to use all the familiar and comprehensive Spring APIs to get your work done. You can chain multiple functions together to offer additional features, and there’s support for multiple inputs and outputs to bring merging, joining, and other more advanced use cases within easy reach. public Function<Flux<String>, Flux<String>> uppercase() {

}

Spring Cloud Data Flow

Spring Cloud Data Flow offers developers a range of tools and automation for working with all kinds of data sources and destinations. Spring Cloud Data Flow helps you to develop, deploy, manage, and scale high-throughput streaming data pipelines across multiple cloud-native platforms. It also features a rich user interface for building and monitoring your applications. Learn more



Spring AMQP and Spring for Apache Kafka

With the Spring AMQP and Spring for Apache Kafka projects, you can apply core Spring concepts to the development of Kafka- or RabbitMQ-based messaging solutions.

Both include "template" as a high-level message handling abstraction, and support message-driven POJOs with a "listener container."
