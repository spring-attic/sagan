The Spring IO Platform

Spring IO brings together the core Spring APIs into a cohesive and versioned foundational platform for Spring applications. On top of this foundation it also provides domain-specific execution environments for a variety of enterprise workloads.

Spring IO Foundation

Spring IO Foundation is a cohesive set of APIs and embeddable runtime components (Spring modules) enabling you to build applications addressing a wide range of enterprise requirements. All of the major workload types are represented: enterprise integration, batch processing, big data processing, web applications (including WebSocket support), and the creation of server-side APIs in support of single-page apps, mobile clients, and services. What all of these workload types have in common is a need to interact with data.

Spring IO Execution

Spring IO Execution provides direct execution support (domain-specific runtimes, or DSRs) for applications built on the IO Foundation modules. A DSR can run standalone without requiring deployment to an external container. The first release of Spring IO includes three DSRs: Spring XD, Spring Boot, and Grails.

Spring XD
